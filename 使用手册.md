## 一句话产品定位

> **给业务用户的 SAP BW 取数副驾驶 —— 用大白话提需求,拿到能直接用的 Excel。**

业务用户**不写 BEx Query、不装 AfO 客户端、不找 IT 排队**，浏览器即用。系统底层用 LLM 自动定位 BW 服务、拼装 OData 查询，结果**填进 Excel** 而非屏幕上的临时表格。

## 现有功能全清单

### 2.1 业务用户能用什么（左侧栏 4 项）

| 入口 | 路由 | 做什么 |
|---|---|---|
| 📊 模板画廊 | `/` | 浏览所有现成报表模板,点卡片填几个参数 → 出 Excel |
| 💬 自由对话 | `/chat` | 没有合适模板时用自然语言提问,LLM 自动找数据源、拼查询、出 Excel |
| 📁 我的任务 | `/tasks` | 查看自己所有历史任务的状态、行数、用时;一键重下载 Excel |
| 🔑 我的 API Keys | `/llm-keys` | 配置自己的 LLM 密钥(每用户独立,不共享) |

**画廊上线时自带 3 个种子模板：**
- `monthly_sales_region` 月度销售大区简报（参数：月份、大区、Top N）
- `top_customers` 客户销售排名（参数：月份、Top N、可选大区）
- `plant_yield` 工厂月度良率（参数：月份）

### 2.5 后端 REST API（27 个端点，除 `/api/status` 与 `/api/auth/login` 外均需 Cookie + JWT）

```
# 系统 (公开,登录页可先调获取 bw_mode / 当前模型)
GET    /api/status                    系统状态(BW模式、LLM、Skills 数;带 cookie 时携带当前用户视角)

# 鉴权
POST   /api/auth/login                登录 (BW Basic Auth 验证)
POST   /api/auth/logout               登出
GET    /api/auth/me                   当前身份

# LLM
GET    /api/llm/models                可选模型 + 当前 + 每个是否 ready
POST   /api/llm/model                 切换全局当前模型
GET    /api/llm/keys                  当前用户的 keys 状态
PUT    /api/llm/keys/{env_var}        配置自己的 key
POST   /api/llm/keys/{env_var}/test   实地拨测 key 是否真有效
DELETE /api/llm/keys/{env_var}        清除自己的 key

# Skills
GET    /api/skills?q=                 列出 / 搜索模板
GET    /api/skills/{id}               模板详情
POST   /api/skills/{id}/run           跑一个模板,出 Excel

# BW 服务（自由模式用）
GET    /api/services?q=               BW 目录(mock/live 共用)
GET    /api/services/{name}           服务的 $metadata 简化版

# Chat (LLM 驱动的自由对话)
POST   /api/chat                      消息 → LLM tool-use → Excel

# 任务
GET    /api/tasks                     我的历史
GET    /api/tasks/{id}                单任务详情
GET    /api/tasks/{id}/file           下载 Excel

# 管理员
GET    /api/audit                     审计日志
GET    /api/sensitive-fields          敏感字段列表
POST   /api/sensitive-fields          新增/更新
DELETE /api/sensitive-fields/{svc}/{f} 删除
POST   /api/admin/skills/reload       热重载 data/skills/
```

📖 完整交互式文档在 http://127.0.0.1:8000/docs （FastAPI 自动生成的 Swagger UI）

## 业务用户：日常使用

### 4.1 登录

1. 浏览器打开 http://127.0.0.1:8000
2. 输入你的 BW 用户名密码（mock 模式可任填）
3. 进入主界面

### 4.2 三种取数方式（按典型场景）

#### 场景 A：每月固定要的报表 → 模板画廊

1. 首页 📊 模板画廊 → 看到「月度销售大区简报」卡片 → 点开
2. 选「月份 = 202605」「大区 = HD」「Top N = 10」
3. 点「✨ 生成 Excel」→ 等 2-5 秒 → 屏幕显示前 30 行 + 下载按钮
4. 点「⬇ 下载」拿 .xlsx

📦 Excel 包含 2 个 sheet：
- 「数据」sheet：表头是中文 label（自动从 BW metadata 取），列宽自适应，数字千分位，冻结首行
- 「查询信息」sheet：原始提问、Skill 版本、OData URL、用时、bw_mode（审计冗余）

#### 场景 B：临时想问个数 → 自由对话

1. 左侧栏点 💬 自由对话
2. 在输入框打字：「上个月华东大区销售情况」
3. 智能体会：
   - 调 `list_skills` 找到 monthly_sales_region 模板
   - 把"上个月"换算成 `202605`、"华东"映射 `HD`
   - 调 `run_skill` 出 Excel
4. 屏幕显示一段简洁分析 + Excel 卡片 + 关键发现
5. 点 Excel 卡片下载

**对话示例**：
- "查 2026 年 5 月销售额最高的 10 个客户"
- "本月各工厂良率,低于 95% 的需重点关注"
- "ZBW_SALES_SRV 服务里都有哪些字段?"

#### 场景 C：以前查过想再查 → 我的任务

1. 左侧栏点 📁 我的任务
2. 表格列出所有历史，按时间倒序
3. 找到那条任务 → 点「⬇ 下载」即可（文件保留在 `data/outputs/`）

### 4.3 配置自己的 LLM Key（**必须**，否则自由对话不可用）

> 模板画廊不需要 LLM key，可以直接用。但「💬 自由对话」需要 LLM 处理自然语言，每个用户必须配自己的 key。

1. 顶栏右上角点 🤖（或左侧栏 🔑 我的 API Keys）
2. 找到你想用的 provider（例如「通义千问 Max」）
3. 点对应 env_var 行的「**配置我的**」
4. 在弹出框粘贴你从对应官方控制台申请的 key:
   - 通义千问 → https://dashscope.console.aliyun.com/apiKey
   - DeepSeek → https://platform.deepseek.com/api_keys
   - Claude → https://console.anthropic.com/settings/keys
   - OpenAI → https://platform.openai.com/api-keys
5. 点「保存」
6. **立即点「🧪 测试」按钮验证 key 真有效**
   - ✅ key 有效 · 耗时 NNNms · 回复 OK → 可以去自由对话了
   - ❌ auth: ... → key 输错了 / 账户被禁
   - ❌ network: ... → 公司防火墙拦了对应域名

🔒 安全说明：
- 你配的 key 只属于你自己，其他用户看不到
- API 永远不回放真值，只回末 4 位
- key 在 SQLite 中 base64 编码（一期），二期升级 AES-256-GCM

### 4.4 切换不同 LLM 模型

顶栏 🤖 下拉 → 点你想用的模型即可。不需要重启。

> 切换是**全局**生效（影响所有用户的当前模型），但 **key 是按用户隔离**。比如你切到 qwen-max，系统会用「你的」DASHSCOPE_API_KEY；隔壁同事切到 deepseek-chat 同时跑，会用「他的」DEEPSEEK_API_KEY。

## 管理员：模板与系统维护

### 5.1 新增 / 修改 Skill 模板

> Skills 由 **管理员维护、所有用户复用**。每个 Skill 就是 `data/skills/<id>/` 目录里的两个文件。

#### 文件夹结构

```
data/skills/
└── <skill_id>/
    ├── SKILL.md         必填,描述 + 参数 + 给 LLM 的指引
    └── service.yaml     必填,数据源 + filter 模板
```

#### SKILL.md 范例

```markdown
---
id: ar_aging_overdue
title: 应收账款逾期清单
description: 列出 90 天以上逾期的客户应收账款。
owner: 财务团队 - 王某
version: 1
keywords: [应收, 账龄, 逾期, AR, overdue]
visible_to: []           # 留空 = 所有人可见;填 ["finance","exec"] 限角色组
params:
  - name: min_amount
    required: false
    default: 50
    description: 仅显示金额 ≥ 这个数(万元)的客户
---

# 应收账款逾期清单

## 适用场景
财务月底催收时拉清单。

## 给 LLM 的指引
- 优先按 Bucket_90Plus 降序
- 总额栏用 TOTAL_F (万元),不要用其他字段
```

#### service.yaml 范例

```yaml
service: ZBW_FIN_SRV
entity_set: ARAging

filter_template: "Bucket_90Plus ge {{ min_amount | default(50) }}"

select:
  - KUNNR
  - CustomerName
  - TOTAL_F
  - Bucket_0_30
  - Bucket_31_60
  - Bucket_61_90
  - Bucket_90Plus

orderby: Bucket_90Plus desc
top: 100
sheet_title: 应收逾期清单
```

#### 发布步骤

1. 在 `data/skills/` 下新建文件夹（id 用英文小写下划线）
2. 写入 `SKILL.md` 和 `service.yaml`
3. 在 UI **🛠 Skill 管理** 页点「🔄 重新加载」（或 `POST /api/admin/skills/reload`）
4. 在右侧详情面板验证字段、模板、参数都正确
5. 通知业务用户即可使用

#### 测试新 Skill

不通过 LLM、不通过 UI：

```powershell
.\.venv\Scripts\python.exe -m cli.chat_bw --run-skill ar_aging_overdue min_amount=80
```

预期：完成 N 行 + Excel 文件路径 + 前 5 行预览。失败时检查 `service.yaml` 字段名是否真的在 BW metadata 里。

### 5.2 审计日志

左侧栏 🔍 审计日志 → 按用户 / 动作过滤。

监控字段：
- `login` / `logout` — 谁何时登录
- `run_skill` — 用了哪个模板，BW 返回多少行
- `chat` — 自然语言提问 + 用了哪个模型 + token 数
- `export` — 谁下载了哪个 Excel
- `switch_model` — 谁切了模型
- `set_api_key` / `delete_api_key` — key 变更（不记录值）

最近 200 条；要长期归档定期把数据库 `audit_log` 表 dump 到冷存储。

### 5.3 敏感字段配置

> 防止把薪资、身份证号之类的字段送进 LLM 上下文（脱敏的实际生效是 M7 内容，目前只支持配置）。

左侧栏 🔒 敏感字段 → 添加 `(service, field, mask_mode)`：
- `redact` → 替换为 `***`
- `partial` → 保留首尾各 2 位
- `hash` → SHA256 前 8 位

### 5.4 监控关键指标

| 指标 | 查询 |
|---|---|
| 当前每个用户的累计 token 消耗 | `SELECT username, SUM(llm_tokens) FROM audit_log WHERE action='chat' GROUP BY username` |
| 哪些 Skill 最常用 | `SELECT skill_id, COUNT(*) FROM tasks WHERE source='skill' GROUP BY skill_id ORDER BY 2 DESC` |
| 失败率 | `SELECT status, COUNT(*) FROM tasks GROUP BY status` |
| 每天活跃用户 | `SELECT DATE(created_at), COUNT(DISTINCT username) FROM audit_log WHERE action='chat' GROUP BY 1` |

> 直接用 SQLite CLI: `sqlite3 data/app.sqlite3 "SELECT ..."`

### 6.4 Mock → Live 字段名漂移修复

Mock 数据用了 BW 现实风格的命名（`NETWR_F`, `0CALMONTH`, `KUNNR`...），但**你公司真实 BW Query 可能命名不同**。比如：
- 净销售额可能叫 `0AMOUNT_F` 而不是 `NETWR_F`
- 月份可能叫 `0CALMONTH` 也可能叫 `ZYRMNTH`

修复方法：

1. 上 UI 「💬 自由对话」问："**ZBW_SALES_SRV 服务有哪些 EntitySet 和字段?**"
2. LLM 调 `get_service_metadata` 把真实字段列出来
3. 对照修改 `data/skills/monthly_sales_region/service.yaml` 里的 `select` / `filter_template` / `orderby`
4. 管理员 → 🛠 Skill 管理 → 🔄 重新加载
5. 重新跑该 Skill 验证

- 接入前：顶栏左侧 ⚠ **MOCK 数据** 橙色角标
- 接入后：顶栏左侧 ● **LIVE** 绿色角标

## 故障排查

### 8.1 决策树

```
我点了 Skill 跑不出 Excel
  ├─ 错误是 "Skill 不存在"   ── 检查 data/skills/<id>/ 文件夹名拼写

  ├─ "authentication" ── key 配错或失效, 上 🔑 我的 API Keys 点 🧪 测试

  ├─ "rate_limit"     ── 触发 provider 限流, 等几秒或换模型
  └─ 其他              ── 看后端日志(uvicorn console)

  ├─ 最大迭代用完 ── LLM_MAX_ITERS 调大,或换 Claude Opus
  ├─ LLM 反复调 list_bw_services ── 提示词不够明确,直接说服务名
  └─ LLM 拼错 OData filter ── 启用上下文,先调 metadata

刚改完 SKILL.md 没生效
  └─ 管理员 → 🛠 Skill 管理 → 🔄 重新加载

UI 刷不出新功能
  ├─ Vite build 跑了吗 ── cd web && npm run build
  └─ 浏览器缓存 ── Ctrl+Shift+R 强制刷新

uvicorn 起不来
```

| 凭据只在内存(cred_cache) | uvicorn 重启后所有用户重登 | 接受;重启不频繁 |
