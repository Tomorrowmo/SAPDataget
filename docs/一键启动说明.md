# 一键启动说明

## 用法
- **双击** `start.bat`（最省事，自动绕过 PowerShell 执行策略限制）；或
- 右键 `start.ps1` →「用 PowerShell 运行」；或在 PowerShell 里执行 `.\start.ps1`。

## 它做了什么
1. `.env` 不存在 → 从 `.env.example` 复制（默认 **mock 模式**，可离线直接跑）。
2. 检查后端依赖（`fastapi/uvicorn/litellm`），缺则 `pip install -r requirements.txt`。
3. `web/node_modules` 不存在 → `npm install`。
4. 在**两个独立窗口**分别启动：
   - 后端 FastAPI：`http://127.0.0.1:8000`
   - 前端 Vite：`http://localhost:5173`
5. 等后端 `/api/status` 就绪 → 自动打开浏览器。用 **admin（空密码）** 登录。

> 装依赖时会自动探测本机代理（v2ray/clash 常用 `7897/10809/10808/7890/1080`）。

## 可选参数
| 参数 | 说明 | 默认 |
|---|---|---|
| `-BackendPort` | 后端端口（须与 `web/vite.config.ts` 的 proxy 目标一致） | `8000` |
| `-FrontendPort` | 前端端口 | `5173` |
| `-Python` | 指定 python 可执行 | `python` |
| `-Proxy` | 装依赖用的代理（覆盖自动探测） | 自动探测 |
| `-NoInstall` | 跳过依赖安装（确认已装好时更快） | 关 |
| `-NoBrowser` | 不自动开浏览器 | 关 |

示例：`./start.ps1 -NoInstall -BackendPort 8000`

## 停止
关闭对应的后端/前端窗口即停止该服务。端口已被占用时脚本会自动跳过启动（避免重复）。

## 真连 SAP / 配 LLM
编辑 `.env`：`BW_MODE=live` + `BW_BASE_URL/BW_CLIENT` 等；LLM key 在网页「🔑 LLM 设置」里按用户配置即可。
