# 按登录用户过滤数据 + 会话历史（抄 DataAgent）实施方案

> 配合 [[feedback-odata-agent-expert]]：SAP 端事实只取证不臆造；[[feedback-workflow-commit-md]]：先 commit、出 md、详细提交。

## 一、查询结果只返回"与登录用户有关的数据"（行级归属过滤）

### 取证依据（不臆造）
- 前端 `web/src/pages/Chat.tsx` 现有 `filterRowsByLoginUser`：按**列名 `UName`（不区分大小写）**匹配登录名，
  归一化规则=大写+去空格。但它**只过滤预览**，Excel 仍是全量 → 这正是要补的洞。
- 真实 `ZBW_QUERY_LIST_SRV/LtResultSet`(报告清单) 含 `UName`(报告归属用户);mock 的该 EntitySet **没有** UName。
- 登录名 = BW/SAP 用户名(用 BW 凭据登录)。

### 设计原则（关键：不得用 mock/live 分支）
**按"结果里是否含归属字段"自适应**，而非按数据源分支：
- 结果行含 `OWNER_FIELD`(默认 `UName`,env 可配) → 只保留 `归一化(行[UName]) == 归一化(登录名)` 的行;
- 不含该字段 → 原样返回(交给 BW Analysis Authorization)。
- mock(无 UName)→ 自动 no-op;live(有 UName)→ 自动裁剪。**同一套逻辑,零 mock 约束。**

### 落点（数据→用户的单一隘口）
- 新增 `app/rowsec.py`：`scope_rows_to_user(rows, owner_field, username) -> (rows, applied)`，复用前端归一化口径。
- `orchestrator.run_skill` / `run_free_query`：拿到 rows 后、出 Excel 前裁剪 → **Excel + 预览**都只含本人数据;
  row_count 用裁剪后的实数;裁剪后为空给"该结果中没有属于你的数据"。
- `agent_stream.execute_odata_query`：样本行喂 LLM 前先裁剪(与脱敏一致,口径统一)。
- 报告清单快捷：经 run_free_query 覆盖;answer 用返回的实数。
- config：`Settings.owner_field`(env `OWNER_FIELD`,默认 `UName`)。
- 前端 `filterRowsByLoginUser` 保留(幂等,后端已裁,作纵深防御)。

> 注:服务端 `$filter UName eq '用户'` 是更优(计数/分页正确)的做法,但 mock 无此列、且不能按模式分支,
> 故本批用"字段存在则后端裁剪"的源无关方案;接真 SAP 确认列名/大小写后可再加服务端注入(分析文档 §0 边界)。

## 二、会话历史（抄 DataAgent Sidebar）

### 现状
- 每个"自由对话" = 一个 `source='chat'` 的 task,messages 已存 `task_messages`;`/chat/:task_id` 已能加载历史。
- 缺：像 DataAgent 那样的**会话列表侧栏**(列出过往会话、点击切换、新建、重命名、删除)。

### 后端
- `GET /api/chat/sessions` → 当前用户的 chat 会话 `[{id,title,created_at,finished_at,status}]`,title=会话标题或首条问题(截断)。
- `PATCH /api/chat/sessions/{id}` → 重命名(tasks 表加 `title` 列,db 一次性迁移 ALTER)。
- 删除复用现有 `DELETE /api/tasks/{id}`。

### 前端（照搬 DataAgent Sidebar 交互）
- Chat 页新增**会话侧栏**(左侧窄栏)：会话列表 + 头像色块(按 id 取色) + 高亮当前 + 新建 + 行内重命名 + 删除确认。
- 点击会话 → `navigate(/chat/:id)`(已自动加载历史);新建 → 清空到 `/chat`;发首条消息后刷新列表。
- 布局：`[会话栏 w-56 | 对话 flex | 工作台 w-80(lg)]`,在外层 App 导航之内。
